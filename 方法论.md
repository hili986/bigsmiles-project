# 核酸序列 BigSMILES 表示方法论

# Methodology: BigSMILES Representation of Nucleic Acid Sequences

---

## 1. 问题陈述 / Problem Statement

BigSMILES（BigSMILES: a structurally-based line notation for describing macromolecules, Lin et al. ACS Cent. Sci. 2019）是一种为**随机共聚物**设计的线性记法。其核心语法——随机对象 `{…}`——专门描述由多种重复单元**随机排列**形成的聚合物链。

然而，在核酸研究中（如适配体、反义寡核苷酸、siRNA），**特定碱基序列**才是决定生物功能的关键。例如，PD-L1 适配体的精确序列为：

```
5'-ACGGGCCACATCAACTCATTGATAGACAATGCGTCCACTGCCCGT-3'  (45 nt)
```

**核心矛盾**：BigSMILES 的随机对象无法区分 `ACGT` 和 `TGCA`——两者生成相同的 BigSMILES 表示。正如 Lin et al. 原文所述，生物聚合物的序列特异性应使用 HELM 等专用记法。

**BigSMILES is a line notation designed for *stochastic* copolymers.** Its core syntax — the stochastic object `{…}` — describes polymer chains composed of randomly arranged repeat units. However, nucleic acid research (aptamers, ASOs, siRNAs) demands **sequence specificity**: the exact base order determines biological function. BigSMILES cannot distinguish `ACGT` from `TGCA` — both produce the same representation.

## 2. 文献综述 / Literature Review

### 2.1 BigSMILES（Lin et al. 2019）

- **原理**: 在 SMILES 基础上引入随机对象 `{…}` 和键描述符 `[$]` / `[>]` / `[<]`
- **优势**: 紧凑、人可读、与 SMILES 兼容
- **局限**: 只能描述聚合物*类别*（如"ssDNA"），不能编码特定碱基序列
- **核酸示例**: 可表示"由 dA/dT/dG/dC 随机排列的 DNA 链"

### 2.2 HELM（Hierarchical Editing Language for Macromolecules）

- **原理**: 专为生物聚合物设计，使用残基级编码
- **优势**: 完美编码特定序列；行业标准（Pistoia Alliance）
- **局限**: 工具生态独立于化学信息学主流（如 RDKit 不原生支持）
- **实测**: RDKit `Chem.MolFromHELM()` 在 2025.09.4 版本返回 `None`，不可用

### 2.3 Full SMILES（原子级完整 SMILES）

- **原理**: 逐核苷酸拼接完整分子的原子级 SMILES 字符串
- **优势**: 编码完整序列；RDKit 可直接解析和渲染；无需额外工具
- **局限**: 字符串极长（45-nt 序列约 1800 字符）；人不可读
- **实测**: 45-nt 序列（约 912 原子）可被 RDKit 正常解析和绘图

### 2.4 PSMILES（Polymer SMILES）

- **原理**: 用 `[*]…[*]` 标记重复单元端点
- **优势**: 与机器学习工具链兼容
- **局限**: 同 BigSMILES，不编码序列顺序

### 2.5 G-BigSMILES（Generalized BigSMILES）

- **原理**: BigSMILES 扩展，支持更复杂的拓扑
- **局限**: 仍基于随机对象，不解决序列特异性问题

## 3. 方法对比表 / Method Comparison

| 特性 | BigSMILES | HELM | Full SMILES | PSMILES | G-BigSMILES |
|------|-----------|------|-------------|---------|-------------|
| **序列特异性** | 否 | 是 | 是 | 否 | 否 |
| **紧凑度** | 高 | 高 | 极低 | 高 | 中 |
| **RDKit 支持** | 需检查器 | 不可用* | 原生 | 部分 | 无 |
| **结构图生成** | 仅重复单元 | 不可用* | 完整分子 | 仅重复单元 | 无 |
| **聚合物类别** | 是 | 是 | 隐含 | 是 | 是 |
| **人可读性** | 好 | 好 | 差 | 好 | 中 |
| **标准化程度** | 学术 | 工业 | 通用 | 学术 | 学术 |

\* RDKit HELM 支持在当前版本实测不可用

## 4. 我们的策略 / Our Strategy: Dual-Representation Hybrid

鉴于没有单一方案能同时满足"聚合物类别描述"和"特定序列编码"两个需求，我们采用**双表示混合策略**：

### 表示 1：BigSMILES（聚合物类别）

用于描述核酸的**聚合物类别**——"这是一条由 4 种脱氧核苷酸组成的 DNA 链"。直接复用本项目示例库中的 13.1（ssDNA）和 13.2（ssRNA）。

```
DNA: O{[>]CC3OC(n1cnc2c(N)ncnc12)CC3OP(=O)(O)O[<],
       [>]CC3OC(n1cc(C)c(=O)[nH]c1=O)CC3OP(=O)(O)O[<],
       [>]CC3OC(n1cnc2c(=O)[nH]c(N)nc12)CC3OP(=O)(O)O[<],
       [>]CC3OC(n1ccc(N)nc1=O)CC3OP(=O)(O)O[<];
       [>]CC3OC(n1cnc2c(N)ncnc12)CC3O,...}
```

### 表示 2：Full SMILES（特定序列）

用于编码核酸的**精确碱基序列**——通过磷酸二酯键逐核苷酸拼接完整原子级 SMILES。这是唯一能同时编码序列并生成结构图的方案。

```
ACGT → OCC3OC(n1cnc2c(N)ncnc12)CC3OP(=O)(O)OCC3OC(n1ccc(N)nc1=O)CC3OP(=O)(O)O
       CC3OC(n1cnc2c(=O)[nH]c(N)nc12)CC3OP(=O)(O)OCC3OC(n1cc(C)c(=O)[nH]c1=O)CC3O
```

### 使用场景

| 场景 | 使用表示 |
|------|---------|
| 聚合物数据库检索 | BigSMILES |
| 文献记录聚合物类别 | BigSMILES |
| 分子结构可视化 | Full SMILES |
| 适配体序列精确记录 | Full SMILES |
| 分子量计算 | Full SMILES |
| 与其他核酸工具对接 | Full SMILES + 原始序列 |

## 5. 核苷酸构建模块 / Nucleotide Building Blocks

### 5.1 DNA 脱氧核苷酸

| 碱基 | 内部核苷酸（含 3',5'-磷酸二酯） | 3' 末端核苷酸 |
|------|-------------------------------|--------------|
| **dA** (腺嘌呤) | `CC3OC(n1cnc2c(N)ncnc12)CC3OP(=O)(O)O` | `CC3OC(n1cnc2c(N)ncnc12)CC3O` |
| **dT** (胸腺嘧啶) | `CC3OC(n1cc(C)c(=O)[nH]c1=O)CC3OP(=O)(O)O` | `CC3OC(n1cc(C)c(=O)[nH]c1=O)CC3O` |
| **dG** (鸟嘌呤) | `CC3OC(n1cnc2c(=O)[nH]c(N)nc12)CC3OP(=O)(O)O` | `CC3OC(n1cnc2c(=O)[nH]c(N)nc12)CC3O` |
| **dC** (胞嘧啶) | `CC3OC(n1ccc(N)nc1=O)CC3OP(=O)(O)O` | `CC3OC(n1ccc(N)nc1=O)CC3O` |

### 5.2 RNA 核糖核苷酸

| 碱基 | 内部核苷酸（含 3',5'-磷酸二酯） | 3' 末端核苷酸 |
|------|-------------------------------|--------------|
| **rA** (腺嘌呤) | `CC3OC(n1cnc2c(N)ncnc12)C(O)C3OP(=O)(O)O` | `CC3OC(n1cnc2c(N)ncnc12)C(O)C3O` |
| **rU** (尿嘧啶) | `CC3OC(n1ccc(=O)[nH]c1=O)C(O)C3OP(=O)(O)O` | `CC3OC(n1ccc(=O)[nH]c1=O)C(O)C3O` |
| **rG** (鸟嘌呤) | `CC3OC(n1cnc2c(=O)[nH]c(N)nc12)C(O)C3OP(=O)(O)O` | `CC3OC(n1cnc2c(=O)[nH]c(N)nc12)C(O)C3O` |
| **rC** (胞嘧啶) | `CC3OC(n1ccc(N)nc1=O)C(O)C3OP(=O)(O)O` | `CC3OC(n1ccc(N)nc1=O)C(O)C3O` |

### 5.3 DNA 与 RNA 的差异

1. **糖基**: DNA 为脱氧核糖（C2' 位为 `CC`），RNA 为核糖（C2' 位为 `C(O)C`，含 2'-OH）
2. **碱基**: DNA 用胸腺嘧啶 T（含 5-甲基 `cc(C)c`），RNA 用尿嘧啶 U（无 5-甲基 `ccc`）
3. **环闭合**: 两者均使用编号 `1`/`2`（碱基杂环）和 `3`（呋喃糖环）

## 6. 磷酸二酯键拼接机制 / Phosphodiester Concatenation Mechanism

### 6.1 拼接原理

核酸链的共价主链由 3',5'-磷酸二酯键连接。在 SMILES 拼接中：

- **5' 端**: 自由羟基 `O`（作为整条链的起始）
- **内部核苷酸**: 含磷酸基团 `P(=O)(O)O`，其 3' 氧与下一核苷酸的 5' 碳相连
- **3' 末端核苷酸**: 以自由 3'-OH `O` 结尾（无尾部磷酸）

### 6.2 拼接算法

```python
full_smiles = "O"                              # 5'-OH 起始
for i, base in enumerate(sequence):
    if i < len(sequence) - 1:
        full_smiles += internal_dict[base]     # 内部核苷酸（含磷酸）
    else:
        full_smiles += terminal_dict[base]     # 3' 末端核苷酸（无磷酸）
```

### 6.3 拼接示例

以 `ACGT` (DNA, 5'→3') 为例：

```
O                                           ← 5'-OH
+ CC3OC(n1cnc2c(N)ncnc12)CC3OP(=O)(O)O    ← dA (内部，含磷酸)
+ CC3OC(n1ccc(N)nc1=O)CC3OP(=O)(O)O       ← dC (内部，含磷酸)
+ CC3OC(n1cnc2c(=O)[nH]c(N)nc12)CC3OP(=O)(O)O  ← dG (内部，含磷酸)
+ CC3OC(n1cc(C)c(=O)[nH]c1=O)CC3O         ← dT (末端，无磷酸)
```

### 6.4 环闭合编号安全性

每个核苷酸片段内部使用环闭合编号 `1`、`2`（碱基杂环）和 `3`（呋喃糖五元环）。这些编号在每个片段内**开闭完成**，不会跨核苷酸冲突。这是该拼接方法可行的关键保证。

### 6.5 方向处理

- **5'→3'（默认）**: 按序列从左到右拼接
- **3'→5'**: 将序列反转后按相同逻辑拼接（等效于反向互补链的 5'→3' 方向）

## 7. 验证与讨论 / Validation and Discussion

### 7.1 RDKit 验证

所有生成的 Full SMILES 均通过 RDKit `Chem.MolFromSmiles()` 验证：

| 序列 | 长度 | SMILES 字符数 | RDKit 解析 | 原子数 |
|------|------|-------------|-----------|-------|
| `A` | 1 nt | ~40 | 通过 | ~22 |
| `ACGT` | 4 nt | ~170 | 通过 | ~80 |
| `ACGTACGTACGT` | 12 nt | ~510 | 通过 | ~240 |
| PD-L1 适配体 (45 nt) | 45 nt | ~1800 | 通过 | ~900 |

### 7.2 BigSMILES 验证

生成的 BigSMILES 字符串通过本项目 `bigsmiles_checker.check_bigsmiles()` 三阶段验证（分词→解析→语义检查），与示例库 13.1/13.2 完全一致。

### 7.3 局限性

1. **Full SMILES 不可读**: 45-nt 序列的 SMILES 约 1800 字符，人类无法直接阅读
2. **渲染极限**: 超长序列（>100 nt）的完整分子图可能过于密集，需使用片段预览
3. **无二级结构**: Full SMILES 仅编码共价结构，不包含碱基配对/折叠信息
4. **无修饰支持**: 当前实现仅支持标准 4 种碱基，不支持修饰碱基（如 m6A、BrdU）
5. **BigSMILES 不含序列**: BigSMILES 表示与输入序列无关，仅反映聚合物类别

### 7.4 未来扩展方向

- 支持修饰碱基（扩展核苷酸字典）
- 支持双链 DNA（两条互补链的拼接）
- 与 HELM 的互转换（待 RDKit HELM 支持成熟）
- 支持环状核酸（如质粒）
